# 控制小车后续校准路线

本文档用于按顺序完成剩余实车校准。已经确认的参数统一保存在
`config/vehicle_calibration.h`，测试程序放在 `tests/`，正式比赛逻辑仍放在
`app/` 和 `control/`。

## 使用规则

- 每次只校准一个变量，其他参数保持不变。
- 每组测试至少重复 5 次，不能用单次最好结果作为最终参数。
- 每次记录日期、电池电压、场地材质、轮胎状态、负载和测试速度。
- 先保存原始 UART 数据，再计算平均值、最大误差和重复性。
- 新参数先写入独立测试程序，实车通过后再更新
  `config/vehicle_calibration.h` 和正式控制层。
- ICM45688 已校准算法和内部参数不修改，只验证安装、振动与温漂表现。
- 测试时保留 FreeRTOS 健康监控、电机命令超时和看门狗保护。

## 推荐执行顺序

```text
机械状态固定
    -> 左右轮速度映射与速度闭环
    -> 有效轮距和编码器转角
    -> 运动中平滑转弯
    -> 数字灰度位置模型
    -> 灰度循线 PID
    -> 加减速、刹车和电压补偿
    -> 完整赛道连续运行
    -> 长时间可靠性验证
```

必须先完成轮速闭环，再调整循线 PID。否则电机左右差异会被错误地吸收到
循线参数中，电量或速度变化后会重新跑偏。

## 阶段 0：固定机械和测试条件

### 需要确认

- [ ] 左右轮胎型号、直径和安装方向一致
- [ ] 轮胎无明显打滑、松动和偏心
- [ ] 灰度传感器安装高度固定
- [ ] IMU 与车体固定牢靠，坐标方向有标记
- [ ] 比赛负载和电池位置固定
- [ ] 测量左右轮实际中心距，记录机械轮距

### 记录内容

| 项目 | 结果 |
| --- | --- |
| 灰度离地高度 | 待填写 |
| 左右轮中心距 | 待填写 |
| 车轮直径 | 待填写 |
| 标准测试电压 | 待填写 |
| 测试地面材质 | 待填写 |

### 完成标准

拆装电池或搬动车辆后，传感器高度和轮距不发生可见变化。机械状态变化后，
必须重新检查后续标定值。

## 阶段 1：左右轮速度映射与闭环

这是后续全部运动控制的基础，优先级最高。

### 测试内容

- [ ] 测量左右电机最低可靠启动 PWM
- [ ] 测量左右电机最低连续运行 PWM
- [ ] 测量 200、250、300、350、400 PWM 的稳态编码器速度
- [ ] 记录每档速度的左右差异和启动时间
- [ ] 分别调整左右轮速度 PID
- [ ] 测试空载、整车落地和比赛负载
- [ ] 测试高、中、低三档电池电压

### 建议串口记录

```text
$SPEED,ms,battery_mv,target_left,target_right,delta_left,delta_right,pwm_left,pwm_right
```

### 需要得到的参数

- 左右电机启动 PWM
- 左右电机前馈表或线性换算
- 左右轮速度 PID 参数
- 最低稳定速度和最大允许比赛速度

### 完成标准

- 每个速度档连续运行时，左右轮稳态误差不超过目标的 3%。
- 目标速度阶跃后无持续振荡。
- 同一档速度重复 5 次的结果稳定。
- 电压变化时编码器闭环仍能维持目标速度。

## 阶段 2：有效轮距与编码器转角

机械尺量轮距不能直接代替实车有效轮距，因为轮胎变形和地面打滑会改变
实际转弯关系。使用 ICM45688 yaw 作为角度基准，编码器作为待校准对象。

### 测试内容

- [ ] 顺时针 90、180、360 度
- [ ] 逆时针 90、180、360 度
- [ ] 每个角度至少重复 5 次
- [ ] 分别记录左右编码器脉冲和最终 yaw
- [ ] 测试低速和比赛转弯速度
- [ ] 比较顺逆时针不对称
- [ ] 测试连续两圈和四圈累计误差

### 建议串口记录

```text
$TURN,dir,target_deg,yaw_deg,left_pulse,right_pulse,time_ms,battery_mv
```

### 需要得到的参数

- 实车有效轮距
- 顺时针左右轮每度脉冲
- 逆时针左右轮每度脉冲
- 顺逆时针补偿比例
- 不同转弯速度下的打滑误差

### 完成标准

- 编码器前馈能够完成目标角度的主体转动。
- 90 度平均误差不超过 1 度，之后可由 IMU 做小量闭环修正。
- 360 度与多圈测试没有持续单方向累计偏差。

## 阶段 3：运动中连续平滑转弯

比赛跑圈不能使用当前停车转向测试中的“刹停、细脉冲、再启动”。运动转弯
采用分层控制：

```text
路径或灰度位置
      -> 目标曲率 / 目标角速度
      -> 左右轮目标速度
      -> 左右轮速度 PID
      -> 电机
```

编码器负责轮速和距离，陀螺仪负责实际角速度与转角验证。

### 测试内容

- [ ] 直线进入固定半径弯道，不停车
- [ ] 顺时针和逆时针分别测试
- [ ] 测试 30、45、60、90 度连续转弯
- [ ] 使用梯形速度曲线进行基准测试
- [ ] 使用 S 曲线逐渐建立左右轮速度差
- [ ] 标定入弯提前距离
- [ ] 标定弯中速度和出弯恢复时间
- [ ] 检查 yaw 角速度是否出现尖峰或反向摆动

### 建议串口记录

```text
$ARC,ms,target_yaw_rate,yaw_rate,left_target,right_target,left_speed,right_speed
```

### 需要得到的参数

- 最大舒适角加速度
- S 曲线加速和减速时间
- 不同车速下的最大允许角速度
- 常用弯道半径对应的左右轮速度比例
- 入弯提前量与出弯恢复速度

### 完成标准

- 进入和离开弯道时车身无明显顿挫。
- 转弯过程中不停车、不反向微调。
- 同一弯道重复 5 次，出口方向和横向位置稳定。
- 转弯完成后能够直接进入下一段直线控制。

## 阶段 4：数字灰度位置模型

七路灰度是数字输入，首先调整传感器模块上的比较阈值，再建立位图到线路
位置的映射。不要直接开始调 PID。

### 硬件阈值测试

- [ ] 在最终安装高度调整每一路电位器
- [ ] 白底时七路状态稳定
- [ ] 黑线经过时对应通道可靠翻转
- [ ] 在场地最亮和最暗位置重复测试
- [ ] 轻微抬起或压低车头时不产生随机跳变

### 位图测试

- [ ] 单路命中：`0001000` 等基本位置
- [ ] 两路同时命中：线位于两个探头之间
- [ ] 三路及以上命中：粗线、停止线或交叉区域
- [ ] 全部未命中：丢线
- [ ] 全部命中：横线或特殊标志
- [ ] 设置连续采样确认和迟滞，过滤单次跳变

### 需要得到的参数

- 黑线有效电平
- 七路位置权重
- 有效位图表和异常位图表
- 连续有效采样次数
- 丢线判断时间
- 横线和交叉线判断条件

### 完成标准

- 静态左右移动黑线时，计算位置单调变化。
- 相同位置重复测量不会在多个远距离位置间跳变。
- 丢线、宽线和普通线能明确区分。

## 阶段 5：灰度循线 PID

### 调整顺序

1. 固定低速，仅调整 P，使车辆能够跟随直线和缓弯。
2. 增加 D，减少左右摆动和入弯过冲。
3. 只有确认存在长期固定偏差时才增加很小的 I。
4. 低速稳定后，再分别标定中速和比赛速度。

### 测试内容

- [ ] 直线循线
- [ ] 缓弯循线
- [ ] 左右连续 S 弯
- [ ] 短暂丢线恢复
- [ ] 不同速度使用不同增益或增益调度
- [ ] 记录最大横向偏差和摆动次数

### 建议串口记录

```text
$LINE,ms,bits,bias,on_line,base_speed,turn,left_target,right_target
```

### 完成标准

- 直线无持续左右摆动。
- 左右弯表现接近，不依赖固定单侧补偿掩盖问题。
- 短时间丢线能够沿最后有效方向恢复。
- 速度提升后仍保持稳定，不出现高频方向切换。

## 阶段 6：加减速、刹车和距离控制

### 测试内容

- [ ] 测量不同目标速度的起步时间和起步距离
- [ ] 测量 coast、brake 两种停止方式
- [ ] 测量不同速度下的刹车距离
- [ ] 测量 50、100、200 cm 的距离闭环
- [ ] 调整减速窗口与最低闭环 PWM
- [ ] 测试直线到转弯、转弯到直线的速度过渡

### 需要得到的参数

- 各速度的加速度和减速度限制
- S 曲线时间
- 刹车距离表
- 距离目标对应的减速窗口
- 停止后编码器超调补偿

### 完成标准

- 距离控制在不同距离下都能稳定停止，而不是只适用于 100 cm。
- 起步和停止不出现明显方向抖动。
- 进入弯道前可以在指定位置平滑降到目标速度。

## 阶段 7：电池电压补偿

### 测试内容

- [ ] 满电、中电量、低电量分别重复轮速测试
- [ ] 记录相同目标速度下所需 PWM
- [ ] 检查低电量时最低启动速度
- [ ] 确认是否只靠速度闭环即可补偿
- [ ] 必要时增加电压前馈、低压限速和低压报警

### 完成标准

- 合理电压范围内，同一速度目标的实际轮速误差保持稳定。
- 低电压不会导致单侧电机先停转并使车辆突然转向。

## 阶段 8：IMU 安装、振动和温漂验证

此阶段不修改已经校准好的 ICM45688 算法和参数。

### 测试内容

- [ ] 确认 IMU 坐标与车体前后左右方向
- [ ] 静止 5、10、20 分钟记录 yaw 漂移
- [ ] 左右电机分别运行时检查振动影响
- [ ] 直线高速运行时检查 yaw 噪声
- [ ] 急加速和刹车时检查静止判断
- [ ] 冷启动与热机后重复 90 度转角

### 完成标准

- 电机振动不会造成明显持续 yaw 漂移。
- 长时间运行后仍能作为编码器转角的验证基准。
- 急加减速不会触发错误的静止锁定。

## 阶段 9：赛道专项与连续跑圈

### 测试内容

- [ ] 起点、终点、停止线和横线识别
- [ ] 直线距离触发转弯
- [ ] 转弯完成后无停顿进入下一段
- [ ] 线路交叉和特殊标志状态机
- [ ] 丢线后的限时恢复与安全停车
- [ ] 单圈误差记录
- [ ] 连续 5 圈、10 圈累计误差
- [ ] 不同起始位置和电量重复测试

### 完成标准

- 状态切换由明确传感器事件或距离条件触发。
- 不依赖长时间阻塞延时完成比赛动作。
- 连续跑圈不出现逐圈增大的固定方向误差。

## 阶段 10：FreeRTOS 与故障恢复

### 测试内容

- [ ] 暂停每个任务，确认健康监控能检测超时
- [ ] 停止刷新电机命令，确认 100 ms 安全停止
- [ ] 模拟 IMU 数据过期
- [ ] 模拟灰度丢线
- [ ] 模拟编码器长时间无脉冲
- [ ] 串口高负载时检查控制周期
- [ ] 连续运行 30 分钟以上
- [ ] 看门狗复位后确认电机默认关闭

### 完成标准

- 任一关键任务卡死不会导致电机无限保持上次输出。
- 遥测和 OLED 刷新不会阻塞控制任务。
- 故障状态和复位原因可以通过 UART 定位。
